Modbus Protocol

Understanding Modbus RTU and Modbus TCP for industrial communication.

What is Modbus?

Modbus is a widely used serial communication protocol developed by Modicon. It is an open protocol used for connecting industrial electronic devices, including PLCs, drives, and instruments. Modbus is simple, reliable, and supported by many devices.

Modbus Variants

  • Modbus RTU – Serial communication (RS-232, RS-485)
  • Modbus ASCII – Serial with ASCII encoding
  • Modbus TCP – Ethernet-based communication

Modbus Addressing

  • Coils (Read/Write) – 0x (digital outputs)
  • Discrete Inputs – 1x (digital inputs)
  • Input Registers – 3x (analog inputs)
  • Holding Registers – 4x (analog outputs, parameters)

Example: Reading Modbus Data

# Reading Holding Register (Address 40001) # PLC: Modbus Master # Device: VFD at Slave ID 1 # Request: Read Holding Register (Function 03) # Address: 40001, Value: 1500 RPM

A PLC reads the speed value from a VFD using Modbus RTU. The PLC sends a request to the VFD (Slave ID 1) to read holding register 40001, which contains the current speed in RPM.
